Introduction

Product Line Engineering (PLE) is a technology for mass customization that derives product instances from a set of core assets. Where applicable, and after an up-front investment, companies obtain time-to-market reduction through a systematic reuse process. Also companies obtain quality improvements through the standardization and homogenization of product family members among other benefits.

There are two main phases:

  • Domain Engineering: Activities related with the definition of the core assets
  • Application Engineering: Activities related with the usage of these core assets.

The current implementation of SPLE in Capella consists in creating a Generic Product. In this Generic Product, the Domain Engineers tag model elements that correspond to a specific feature (filtering point expressing optionality). Features are abstract characteristics of a system that are relevant for some stakeholder. Then, Application Engineers can derive Specific Products containing only a selected set of features. This selection of features for extracting a specific product is called FilteringResult. The specific product will contain the elements associated to the selected features in addition to the common part that is shared among all products in the product family.
